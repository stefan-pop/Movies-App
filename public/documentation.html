Request URL Request Body Response Body
GET /movies
Return a list of ALL movies to the user 		No data expected Returns an array of movie objects in JSON format
GET /movies/[title]
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user
The URL's parameters must contain the following data:
* [title] - The title of a movie.
 No data expected Returns a JSON object containing the movie with the title of the URL's parameter.
If [title] = movie_5 , the result will be like follow: 

                    {
                        "title": "movie_5",
                        "genre": "comedy",
                        "description": "The description of movie_5",
                        "image": "url to the image of movie_5",
                        "featured": true,
                        "director": {
                            "name": "Director5",
                            "birthYear": "1960",
                            "deathYear": "",
                            "biography": "The biography of the director"
                        }
                    }
GET /movies/genre/[genre]
Return data about a genre (description) by name/title (e.g., “Thriller”)
The URL's parameters must contain the following data:
* [genre] - The name of a genre. (ex: horror, adventure, comedy, thriller...)
 No data expected Returns a JSON object/array of objects containing the movie/movies belonging to the URL's parameter.
If [genre] = horror , the result will be like follow: 

                    {
                        "title": "movie_5",
                        "genre": "horror",
                        "description": "The description of movie_5",
                        "image": "url to the image of movie_5",
                        "featured": true,
                        "director": {
                            "name": "Director5",
                            "birthYear": "1960",
                            "deathYear": "",
                            "biography": "The biography of the director"
                        }
                    }
GET /movies/details/directors
Return a list of ALL directors. 		No data expected Returns a JSON array of objects containing all the directors and their description like follow: 

                    [
                        {
                            "name": "Director1",
                            "birthYear": "1960",
                            "deathYear": "",
                            "biography": "The biography of the director"
                        },
                        {
                            "name": "Director2",
                            "birthYear": "1964",
                            "deathYear": "",
                            "biography": "The biography of the director"
                        }
                    ]
GET /movies/details/directors/[director]
Return data about a director (name, biography, birth year, death year) by name.
The URL's parameters must contain the following data:
* [director] - The name of a director.
 No data expected Returns a JSON object containing data about a director, by director's name.
If [director] = Director1 , the result will be like follow: 

                    {
                        "name": "Director1",
                        "birthYear": "director's birthyear",
                        "deathYear": "director's deathyear",
                        "biography": "The biography of the director"
                    }
POST /users
Allow new users to register. 		A JSON object with at least the name, username and the email of the user like follow: 

                    {
                        "name": "user_name",
                        "username": "users_username"
                        "email": "example@gmail.com"
                    }
Returns a JSON object containing data from the request body plus an automatically generated ID. 

                    {
                        "name": "user_name",
                        "username": "user_username"
                        "email": "example@gmail.com"
                        "id": "this was generated by uuid express module"
                    }
PUT /users/[id]/[new_username]
Allow users to update their username.
The URL's parameters must contain the following data:
* [id] - the id of the user
* [new_username] - a new username to replace the existing one. 		No data expected. Returns a confirmation message with the modified data ( the previous and the actual username ).
POST /users/[user]/favorites
Allow users to add a movie to their list of favorites. 		A JSON object with the favorite movie. Returns a confirmation message.
DELETE /users/[user]/favorites/[movie_title]
Allow users to remove a movie from their list of favorites.
The URL's parameters must contain the following data:
* [user] - the name of the user.
* [movie-title] - the title of the movie to be removed. 		No data expected. Returns a confirmation message.
DELETE /users/[user]
Allow existing users to deregister.
The URL's parameters must contain the following data:
* [user] - the name of the user.
No data expected. Returns a confirmation message.
